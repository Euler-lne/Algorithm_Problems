以下是 **Lomuto 分区** 与 **Hoare 分区** 的对比笔记，用于快速排序中的划分步骤。

---

## 一、Lomuto 分区

### 核心思想
- 选择最右侧（或最左侧）元素作为基准值（pivot）。
- 维护一个指针 `i`，指向已处理的小于 pivot 的区域的末尾。
- 遍历数组，将小于 pivot 的元素交换到前部，最后将 pivot 放到正确位置。
- 返回 pivot 的最终下标。

### 代码（C++，左闭右闭区间）
```cpp
int lomutoPartition(int arr[], int left, int right) {
    int pivot = arr[right];          // 选最右边为基准
    int i = left - 1;                // i 指向小于 pivot 的最后一个位置
    for (int j = left; j < right; j++) {
        if (arr[j] < pivot) {
            i++;
            swap(arr[i], arr[j]);
        }
    }
    swap(arr[i + 1], arr[right]);    // 将 pivot 放到中间
    return i + 1;                    // pivot 的最终位置
}
```

### 特点
- **简单易懂**：只需一次遍历，逻辑清晰。
- **稳定性**：非稳定，但交换次数相对少（不过实际比 Hoare 多）。
- **性能**：当数组已有序时，分区极不平衡（最坏 O(n²)），但平均 O(n log n)。
- **递归区间**：`[left, p-1]` 和 `[p+1, right]`。

## 二、Hoare 分区

### 核心思想
- 选取中间位置的元素作为基准值（pivot）。
- 两个指针 `i`（左）、`j`（右）相向移动。
- `i` 找到大于等于 pivot 的元素，`j` 找到小于等于 pivot 的元素，交换。
- 直到 `i >= j`，此时 pivot 已经位于某个位置，但最终返回 `j`（左半区结尾）。
- 注意：pivot 本身可能被移动，最终不在其正确位置。

### 代码（C++，左闭右闭区间）
```cpp
int hoarePartition(int arr[], int left, int right) {
    int pivot = arr[left + (right - left) / 2];
    int i = left, j = right;
    while (i <= j) {
        while (arr[i] < pivot) i++;
        while (arr[j] > pivot) j--;
        if (i <= j) {
            swap(arr[i], arr[j]);
            i++;
            j--;
        }
    }
    return j;   // 返回左半区的右边界
    // 结束之后 所有 output[left ... j] 都 ≤ mid。
    //         所有 output[i ... right-1] 都 ≥ mid。
}
```

### 特点
- **效率高**：平均交换次数比 Lomuto 少约 3 倍。
- **不平衡风险较小**：对于已有序数组，依然能较好地划分（因为 pivot 取中间值）。
- **不保证 pivot 在最终位置**：递归时需要用 `j` 和 `j+1` 作为分割点。
- **递归区间**：`[left, j]` 和 `[j+1, right]`。

## 三、Lomuto vs Hoare 对比表

| 特性             | Lomuto 分区                                       | Hoare 分区                                      |
| ---------------- | ------------------------------------------------- | ----------------------------------------------- |
| 基准选取         | 通常选最右（或最左）                              | 通常选中间位置                                  |
| 指针移动方向     | 单向（左到右）                                    | 双向（左→右，右→左）                            |
| 交换次数（平均） | 较多                                              | 较少（约 1/3 的 Lomuto）                        |
| 稳定性           | 非稳定                                            | 非稳定                                          |
| 代码复杂度       | 简单，易理解                                      | 稍复杂，需注意边界                              |
| 对有序数组的处理 | 极差（退化为 O(n²)）                              | 较好（仍能平衡划分）                            |
| 返回值的含义     | pivot 的最终下标                                  | 左半部分的最后一个下标                          |
| 递归调用方式     | `quickSort(left, p-1)`<br>`quickSort(p+1, right)` | `quickSort(left, j)`<br>`quickSort(j+1, right)` |
| 常见应用场景     | 教学、简单实现、对随机数据足够                    | 生产环境、追求性能的快速排序                    |

---

## 四、总结与建议

- **学习阶段**：推荐先用 **Lomuto**，逻辑简单，容易写出正确的快速排序。
- **实际项目**：优先使用 **Hoare** 分区，性能更好，尤其处理重复元素或有序数据时表现更佳。
- **C++ std::sort**：结合了多种优化（如内省排序、三点取中、插入排序等），不是单纯 Hoare 或 Lomuto。

> 注：两种分区都可以通过随机选取 pivot 或三点取中来避免最坏情况。
