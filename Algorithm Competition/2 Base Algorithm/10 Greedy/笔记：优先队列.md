## C++ priority_queue 自定义比较完整笔记（含运算符重载）

### 一、基本公式

```cpp
priority_queue<类型, 容器, 比较类> pq;
```

- 容器一般用 `vector<类型>`（可省略）
- 比较类决定**谁在堆顶**（优先级最高）

---

### 二、默认行为（大顶堆）

```cpp
priority_queue<int> pq;   // 等价于 priority_queue<int, vector<int>, less<int>>
```
- `less<int>` 表示：`a < b` 时，a 排在 b 后面 → 大的在堆顶

---

### 三、小顶堆（用 greater）

```cpp
priority_queue<int, vector<int>, greater<int>> pq;
```
- `greater<int>` 表示：`a > b` 时，a 排在 b 后面 → 小的在堆顶

---

### 四、自定义类型：方法1 – 重载 `operator<`（推荐入门）

在结构体内部重载 `<` 运算符，`priority_queue` 默认使用 `less<T>`，它会调用你的 `operator<`。

```cpp
struct Student {
    string name;
    int score;
    
    // 重载 < 运算符，定义“优先级更低”的规则
    bool operator<(const Student& other) const {
        return score < other.score;   // 分数低的优先级低 → 大顶堆（高分在顶）
    }
};

priority_queue<Student> pq;   // 自动使用重载的 <
```

**如果想实现小顶堆（低分在顶）**，只需反转比较方向：

```cpp
bool operator<(const Student& other) const {
    return score > other.score;   // 分数高的优先级低 → 小顶堆（低分在顶）
}
```

**注意**：重载 `<` 会影响所有使用 `less<Student>` 的地方，包括 `sort` 等，可能产生副作用。但对于简单的优先队列场景，这是最直接的方法。

---

### 五、自定义类型：方法2 – 自定义仿函数（推荐通用）

写一个结构体，重载 `operator()`，作为比较类。

```cpp
struct CompareScoreDesc {   // 大顶堆（高分在顶）
    bool operator()(const Student& a, const Student& b) const {
        return a.score < b.score;
    }
};

priority_queue<Student, vector<Student>, CompareScoreDesc> pq;
```

小顶堆（低分在顶）：
```cpp
struct CompareScoreAsc {
    bool operator()(const Student& a, const Student& b) const {
        return a.score > b.score;
    }
};
```

---

### 六、自定义类型：方法3 – Lambda 表达式（C++11）

```cpp
auto cmp = [](const Student& a, const Student& b) {
    return a.score < b.score;   // 大顶堆
};
priority_queue<Student, vector<Student>, decltype(cmp)> pq(cmp);
```

## 🧠 快速记忆口诀（核心）

> **尖括号里写比较，返回真就往后靠。**  
> **想大顶，用小于；想小顶，用大于。**  
> **八字诀：小于大顶，大于小顶**

### 针对运算符重载的特殊口诀

> **重载小于号，队列自己跑。**  
> **分数小于放右边，高分自然在顶面。**  
> **若想小顶在顶端，大于号里写判断。**

解释：
- 重载 `operator<` 时，`return score < other.score` → 大顶堆（高分在顶）
- `return score > other.score` → 小顶堆（低分在顶）

---

## 七、对比记忆（sort vs priority_queue）

| 功能 | sort 的 comp | priority_queue 的 comp |
| ---- | ------------ | ---------------------- |
| 升序 | `a < b`      | 小顶堆：`a > b`        |
| 降序 | `a > b`      | 大顶堆：`a < b`        |

**口诀**：  
> **sort 同向，queue 反向。**  
> 想升序，sort 用小于，queue 用大于。  
> 想降序，sort 用大于，queue 用小于。

---

## 八、总结一句话

- **重载 `<`**：`return a.field < b.field` → 大顶堆；`return a.field > b.field` → 小顶堆。
- **仿函数**：`operator()` 返回 `a.field < b.field` → 大顶堆；返回 `a.field > b.field` → 小顶堆。
- **记忆口诀**：**小于大，大于小**（比较里写小于，得到大顶堆；写大于，得到小顶堆）。

把这些口诀记住，优先队列的自定义比较永远不会混淆。
