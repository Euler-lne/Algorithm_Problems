# 笔记：Game Prediction（田忌赛马变种）

## 1. 问题重述

- 有 **m** 个人（包括你），每人 **n** 张牌，所有牌点数 **1 ~ m×n** 且互不相同。
- 进行 **n** 轮，每轮每人出一张牌，点数最大者赢该轮。
- 给定你手中的 n 张牌（其余牌归对手，他们可合作），求 **在最坏情况下（对手尽力让你少赢）你至少能赢多少轮**。

## 2. 核心贪心思想（田忌赛马）

> 对手会尽量用自己最大的牌来压你较大的牌，让你输。  
> 你则要安排出牌顺序，使赢的轮数最多。

**贪心策略**（从大到小处理所有牌）：

1. 将你的牌标记出来。
2. 从最大点数 `total = m×n` 向下扫描到 1。
3. 维护一个计数器 `cnt`：表示当前**比正在扫描的牌大且属于对手的牌的数量**（即对手可用于压制你当前牌的储备）。
   - 遇到**对手的牌** → `cnt++`（对手多了一张大牌储备）
   - 遇到**你的牌**：
     - 若 `cnt > 0` → 对手有更大牌压你，你输，并且消耗一张对手大牌：`cnt--`
     - 若 `cnt == 0` → 没有对手牌比你大，你赢：`ans++`

**最终 `ans` 即为所求。**

## 3. 为什么这样是对的？

- 从大到小扫描，模拟了你从最大牌开始出的顺序（最优出法）。
- `cnt` 精确记录了对手手里“比当前牌大”的牌数，这些牌可以按需用来压你。
- 该贪心满足**贪心选择性质**和**最优子结构**，已被数学证明是最优的。

## 4. 代码实现（C++）

```cpp
#include <cstdio>
#include <cstring>

int main() {
    int m, n, caseNum = 0;
    while (scanf("%d%d", &m, &n) == 2 && (m || n)) {
        int total = m * n;
        bool my[1005] = {false};   // 标记自己的牌
        for (int i = 0; i < n; ++i) {
            int x;
            scanf("%d", &x);
            my[x] = true;
        }

        int cnt = 0, win = 0;
        for (int i = total; i >= 1; --i) {
            if (my[i]) {          // 这张牌是我的
                if (cnt == 0) ++win;
                else --cnt;
            } else {              // 这张牌是对手的
                ++cnt;
            }
        }
        printf("Case %d: %d\n", ++caseNum, win);
    }
    return 0;
}
```

## 5. 样例推演（以第一组为例）

m=2, n=5, total=10  
你的牌：1,7,2,10,9 （标记为真）

从10到1扫描：

| 点数 | 谁有？ | cnt（前） | 动作        | cnt（后） | win |
| ---- | ------ | --------- | ----------- | --------- | --- |
| 10   | 你     | 0         | 赢          | 0         | 1   |
| 9    | 你     | 0         | 赢          | 0         | 2   |
| 8    | 对手   | 0         | cnt++       | 1         | 2   |
| 7    | 你     | 1         | 输（被8压） | 0         | 2   |
| 6    | 对手   | 0         | cnt++       | 1         | 2   |
| 5    | 对手   | 1         | cnt++       | 2         | 2   |
| 4    | 对手   | 2         | cnt++       | 3         | 2   |
| 3    | 对手   | 3         | cnt++       | 4         | 2   |
| 2    | 你     | 4         | 输          | 3         | 2   |
| 1    | 你     | 3         | 输          | 2         | 2   |

结果 win = 2 ✅

## 6. 易错点与注意事项

- **总牌数** = m × n，不要漏乘。
- 数组大小至少为 total+1（点数从1开始）。
- 对手的牌不需要显式存储，只需标记自己的牌。
- 扫描方向必须**从大到小**，才能模拟“用大牌压大牌”的过程。
- `cnt` 不会超过对手的总牌数，无需额外处理。

## 7. 复杂度分析

- 时间：O(m×n)，只需一次遍历。
- 空间：O(m×n) 用于标记数组，最大 20×50=1000，非常小。

## 8. 总结

> 这道题的精髓在于 **“从大到小扫描 + 计数对手剩余大牌”** 的贪心技巧。  
> 它等价于田忌赛马中“用自己最小的牌去输掉对手最大的牌”的思想，但实现上更简洁优雅。

**一句话记忆**：从大到小看牌，遇到对手的就存起来，遇到自己的就看看有没有存的大牌能压你，有则输，无则赢。
