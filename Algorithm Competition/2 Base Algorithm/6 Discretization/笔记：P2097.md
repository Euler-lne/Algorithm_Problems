## 一、题目分析

**题意**  
有 n 台电脑，m 条双向数据线。若一台电脑获得数据，它可以通过数据线将数据传给所有直接或间接相连的电脑。问至少需要将数据输入几台电脑，才能使所有电脑都获得数据。

**本质**  
每台电脑是一个节点，每条数据线是一条无向边。数据传播的过程等同于从已输入的电脑出发，沿着边遍历整个连通分量。因此，**至少输入的电脑数 = 图中的连通分量个数**。每个连通分量只需输入一次，数据就能覆盖该分量内所有电脑。

## 二、原代码错误分析
  ```cpp
#include <algorithm>
#include <cstdio>
#include <vector>

using namespace std;

bool is_contain(vector<vector<int>> &g, int a, int b)
{
    if (g[a].size() == 0)
        return false;
    for (auto t : g[a]) {
        if (t == b)
            return true;
    }
    return false;
}

int main()
{
    int n, m;
    scanf("%d%d", &n, &m);
    vector<vector<int>> g(n + 2);
    vector<int> mp(n + 1);
    for (int i = 0; i < m; i++) {
        int a, b;
        scanf("%d%d", &a, &b);
        if (is_contain(g, a, b))
            continue;
        if (g[a].size() == 0)
            g[a].push_back(a);
        if (g[b].size() == 0)
            g[b].push_back(b);
        g[a].push_back(b);
        g[b].push_back(a);
    }
    sort(g.begin(), g.end(), [&](vector<int> x, vector<int> y) {
        return x.size() > y.size();
    });
    int ans = 0;
    for (int i = 0; i < n; i++) {
        mp[g[i][0]] = i;
    }
    for (int i = 0; i < n; i++) {
        if (mp[g[i][0]] == -1)
            continue;
        ans++;
        for (auto t : g[i]) {
            mp[t] = -1;
        }
    }
    printf("%d", ans);
    return 0;
}
  ```

原代码（见题目描述）试图用一种贪心策略解决，但存在多处错误，导致运行错误（RE）和答案错误（WA）。

### 1. 邻接表处理错误
- 在存储边时，如果 `g[a].size() == 0`，代码会先向 `g[a]` 加入 `a` 自身；同样对 `g[b]` 也加入自身。  
  **问题**：这导致每个节点的邻接表至少包含自己，但后续逻辑中，排序后直接取 `g[i][0]` 作为节点编号。当某个节点根本没有边时，`g[i]` 中只有它自己，这样做虽然不会越界，但破坏了算法本意。
- 重边处理：`is_contain` 函数只是线性查找，效率低且容易漏判（因为已经先加入了自身，再检查是否已有 `b`，实际上可能错误）。

### 2. 排序与后续处理混乱
- `sort(g.begin(), g.end(), ...)` 将邻接表按大小（度数）降序排列。排序后，`g[i]` 不再是原来对应节点 i 的邻接表，而是按度数排序后的某个节点的邻接表。  
- 随后用 `mp[g[i][0]] = i` 记录节点编号到排序后位置的映射，但 `g[i][0]` 可能等于 0 或别的值？实际上 `g` 的大小是 `n+2`，前 `n` 个有效位置（索引 0~n-1）存放的是各节点的邻接表，但排序后这些表可能来自任意节点。
- 在标记时，用 `mp[t] = -1` 试图标记已访问节点，但 `mp` 的下标是节点编号，而 `t` 是邻接表中的节点，这没有问题，但排序后 `g[i]` 已经与原始节点错位，导致逻辑混乱。

### 3. 算法本质错误
- 代码试图通过“每次选择当前度数最大的节点，将其及其所有邻接点标记为已处理”来求最少输入数量。这是典型的**最小支配集**贪心，但本题要求的是连通分量数，两者完全不同。  
  例如一个链状图（1-2-3-4），连通分量数为 1，但该贪心可能选择节点 2 和 4 等，得出错误答案。

### 4. 数组越界（RE）
- `mp` 大小为 `n+1`，但排序后 `g[i][0]` 可能为 0（因为 `g` 有 `n+2` 个元素，末尾有额外空表），访问 `mp[0]` 导致越界。  
- 当 `g[i]` 为空表时，直接访问 `g[i][0]` 也会导致 RE。

## 三、正确解法（并查集）

### 思路
- 使用并查集维护节点间的连通关系。初始时每个节点单独一个集合。
- 对于每条边 (u, v)，将 u 和 v 所在的集合合并。
- 处理完所有边后，统计有多少个不同的根节点（即集合数），答案即为集合数。

### 步骤
1. 读入 n, m。
2. 初始化并查集：`parent[i] = i`。
3. 对于每条边 (a, b)，调用 `union(a, b)`。
4. 遍历 1..n，找到每个节点的根 `find(i)`，用布尔数组记录根是否出现过。
5. 统计不同根的个数并输出。

### 处理重边和自环
- 并查集合并操作是幂等的，多次合并同一对节点不影响结果。
- 自环（a == b）对连通性无影响，可以忽略或正常合并（合并自己无影响）。

### 时间复杂度
- 接近 O((n+m) α(n))，其中 α 是阿克曼函数的反函数，可视为常数，非常高效。

## 四、并查集原理

### 定义
并查集（Disjoint Set Union, DSU）是一种树型数据结构，用于处理一些不相交集合的合并及查询问题。它支持两种操作：
- **查找（Find）**：确定某个元素属于哪个集合（返回代表元素）。
- **合并（Union）**：将两个不同集合合并成一个。

### 实现要点
- **数组 `parent[x]`**：存储 x 的父节点。根节点的父节点指向自身。
- **路径压缩**：在查找过程中，将沿途节点的父节点直接指向根节点，加速后续查找。
  ```cpp
  int find(int x) {
      if (parent[x] != x)
          parent[x] = find(parent[x]); // 路径压缩
      return parent[x];
  }
  ```
- **按秩合并**：合并时，将“秩”（如树高或大小）较小的树接到较大的树上，避免树退化为链表。常用方法：维护 `size` 数组，将小集合合并到大集合。

### 在本问题中的应用
- 初始化：每个节点独立成集合。
- 每读入一条边 (u, v)，调用 `union(u, v)` 将两个节点所在集合合并。
- 最终，同一连通分量中的所有节点属于同一个集合。
- 统计集合数即可得到连通分量个数。

## 五、正确代码（并查集）

```cpp
#include <iostream>
#include <vector>
using namespace std;

vector<int> parent;

int find(int x) {
    if (parent[x] != x)
        parent[x] = find(parent[x]); // 路径压缩
    return parent[x];
}

void unite(int a, int b) {
    int ra = find(a), rb = find(b);
    if (ra != rb)
        parent[ra] = rb; // 简单合并，未按秩优化也可通过本题
}

int main() {
    int n, m;
    scanf("%d%d", &n, &m);

    parent.resize(n + 1);
    for (int i = 1; i <= n; ++i)
        parent[i] = i;

    for (int i = 0; i < m; ++i) {
        int a, b;
        scanf("%d%d", &a, &b);
        unite(a, b);
    }

    vector<bool> isRoot(n + 1, false);
    for (int i = 1; i <= n; ++i) {
        isRoot[find(i)] = true;
    }

    int ans = 0;
    for (int i = 1; i <= n; ++i) {
        if (isRoot[i]) ans++;
    }
    printf("%d\n", ans);

    return 0;
}
```

## 六、对比总结

| 项目       | 原代码（错误）                          | 正确解法         |
| ---------- | --------------------------------------- | ---------------- |
| 核心思想   | 贪心选最大度节点，试图最小化覆盖        | 统计连通分量个数 |
| 数据结构   | 邻接表，排序，额外数组 mp               | 并查集           |
| 时间复杂度 | O(m log m + n^2) 左右（因多次线性查找） | O((n+m) α(n))    |
| 空间复杂度 | O(n+m)                                  | O(n)             |
| 正确性     | 错误，无法保证连通分量数                | 正确             |
| 鲁棒性     | 可能越界 RE                             | 无越界风险       |

---

**总结**：本题的正确解法是统计连通分量数量，而非复杂贪心。并查集是解决此类问题的利器，简单高效。原代码的错误在于对题意理解偏差和实现细节上的致命缺陷。
