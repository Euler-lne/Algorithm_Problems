# 离散化区间选择详解：一维左闭右开 vs 二维闭区间

在区间覆盖问题中，离散化是常用的技巧。对于一维（整数点）和二维（连续区域），由于问题的本质不同，离散化时对区间的表示方法也有所区别。下面详细分析两者的原因。

---

## 一、一维情况：Color the Ball

### 问题本质
- 球的位置是**整数点**，如 1, 2, 3, …。
- 操作是涂色一段连续的整数区间 `[a, b]`（包含端点）。
- 需要找到最长的连续白色整数段。

### 为什么用左闭右开 `[L, R+1)`？

#### 1. 离散化的目标
我们希望将无限个整数点压缩为有限个**段**，每个段对应一段连续整数区间。离散化点应该是所有操作的端点以及可能需要的边界。

#### 2. 边界处理的简洁性
如果直接使用闭区间 `[L, R]` 进行离散化：
- 收集所有 `L` 和 `R`。
- 排序去重后得到数组 `p`。
- 相邻点 `p[i]` 和 `p[i+1]` 之间的整数区间是 `[p[i], p[i+1]]`，长度 = `p[i+1] - p[i] + 1`。
- 当两个操作相邻时，例如 `[1,4]` 和 `[5,8]`，离散化点 1,4,5,8，段 `[1,4]` 和 `[5,8]` 之间没有空隙（因为 4+1=5），但若中间有空隙（如 `[1,2]` 和 `[4,5]`），段 `[1,2]` 和 `[4,5]` 之间会有一个整数 3 未被任何段覆盖，无法单独表示。此时要处理部分覆盖（如只涂色 [2,3]），就需要拆分段，实现复杂。

#### 3. 左闭右开的优势
改用左闭右开 `[L, R+1)` 后：
- 每个操作覆盖的是 `[L, R+1)` 内的所有整数。
- 收集所有 `L` 和 `R+1`。
- 排序去重后得到数组 `p`。
- 相邻点 `p[i]` 和 `p[i+1]` 之间的区间 `[p[i], p[i+1])` 恰好对应一段连续整数 `p[i], p[i+1]-1`，长度 = `p[i+1] - p[i]`（无需 +1）。
- 如果两个操作之间有空隙，例如 `[1,2]` 和 `[4,5]`，左闭右开表示为 `[1,3)` 和 `[4,6)`，离散化点 1,3,4,6，段 `[1,3)` 和 `[4,6)` 之间有空隙 `[3,4)`，这个空隙正好对应整数 3 没有被任何操作覆盖，可以独立表示。这样任何子区间都可以由若干完整段组成，无需拆分段。
- 涂色时，对于操作 `[L, R+1)`，找到 `left = mp[L]`, `right = mp[R+1]`，直接覆盖 `[left, right)` 的所有段，因为边界一定与段边界对齐。

#### 4. 长度计算与输出
- 统计白色段时，累加每个段长度 `p[i+1] - p[i]`。
- 白色段右端点 = `p[i+1] - 1`（因为段覆盖到 `p[i+1]-1`）。
- 无需额外加减，避免出错。

#### 5. 示例
操作 `[1,4]` 和 `[3,5]`（闭区间）。
- 左闭右开：`[1,5)` 和 `[3,6)`。
- 离散化点：1,3,5,6。
- 段0: `[1,3)` → 整数1,2
- 段1: `[3,5)` → 整数3,4
- 段2: `[5,6)` → 整数5
覆盖后，段0白色，段1先白后黑（黑色覆盖），段2黑色。白色段只有段0，长度2，左=1，右=2。正确。

## 二、二维情况：City Planning

### 问题本质
- 矩形覆盖的是**连续平面区域**，由左下角 `(x1, y1)` 和右上角 `(x2, y2)` 定义，包含边界。
- 坐标是整数，但区域是连续的（面积连续）。
- 后建的矩形会覆盖重叠部分。
- 目标是最大化总价值（面积 × 单位价值）。

### 为什么直接用闭区间 `[x1, x2] × [y1, y2]`？

#### 1. 连续区域的特性
- 边界线面积为 0。两个矩形共享一条边时，重叠的边界线不会产生额外面积。
- 因此，用闭区间离散化时，相邻格子共享边界线，但面积计算不会重复，因为宽度/高度是差值，不包含边界线宽度。

#### 2. 离散化的实现
- 收集所有矩形的 `x1, x2` 和 `y1, y2`。
- 排序去重后得到数组 `xs` 和 `ys`。
- 相邻点 `xs[i]` 和 `xs[i+1]` 之间的区间是 `[xs[i], xs[i+1]]`，宽度 = `xs[i+1] - xs[i]`。注意，这个宽度恰好是整数差，因为坐标是整数，但区间是连续的，所以宽度就是差值。
- 同理，y 方向高度 = `ys[j+1] - ys[j]`。
- 每个小格子 `[xs[i], xs[i+1]] × [ys[j], ys[j+1]]` 的面积 = `(xs[i+1]-xs[i]) * (ys[j+1]-ys[j])`，直接计算，无需 +1。

#### 3. 覆盖操作的索引
- 对于矩形 `[x1, x2] × [y1, y2]`，找到 `x1` 在 `xs` 中的位置 `xl`，`x2` 的位置 `xr`（注意 `x2` 在数组中存在）。
- 覆盖时，需要覆盖 x 方向从 `xl` 到 `xr-1` 的格子（因为格子由相邻 x 坐标对定义），即使用左闭右开索引 `[xl, xr)`。
- 同理 y 方向 `[yl, yr)`。
- 这样，每个矩形正好覆盖一组完整的格子，不会部分覆盖（因为所有矩形的边界都落在离散化点上）。

#### 4. 排序覆盖法的正确性
- 将矩形按价值升序排序，然后依次覆盖（直接赋值）。由于后覆盖的矩形价值更大，每个格子最终得到覆盖它的最大价值。
- 这是因为题目允许任意安排建造顺序，而排序后价值大的后建，正好模拟了最优策略（每个格子由价值最大的矩形覆盖）。
- 因此，无需复杂 DP，直接计算即可。

#### 5. 与左闭右开的等价性
- 如果改用左闭右开 `[x1, x2+1) × [y1, y2+1)`，离散化点包含 `x2+1` 和 `y2+1`，格子面积为 `(xs[i+1]-xs[i])*(ys[j+1]-ys[j])`，与闭区间版本完全相同，只是坐标平移。
- 但闭区间版本更直观，因为输入直接给出了闭区间，无需转换。

## 三、为什么一维不能直接用闭区间？

### 主要区别
- 一维覆盖的是整数点，相邻整数点之间没有间隙，但离散化后若用闭区间，相邻段之间可能有重叠或缝隙，且长度计算需要 +1。
- 更重要的是，当操作部分覆盖一个段时（例如只覆盖整数 2-3 而段是 1-4），闭区间表示需要拆分段，而左闭右开表示中，所有操作边界都落在离散化点上，因此永远不会出现部分覆盖，简化了实现。

### 一维闭区间的尝试与失败示例
假设我们坚持用闭区间离散化一维：
- 收集所有 `L` 和 `R`，排序去重得到 `p`。
- 定义段为 `[p[i], p[i+1]]`（相邻段共享端点）。

考虑两个操作：`[1,4] w` 和 `[3,5] b`。
1. 收集端点：`{1,4,3,5}` → 排序去重得 `p = [1,3,4,5]`。
2. 段定义：
   - 段0: `[1,3]` → 整数 {1,2,3}
   - 段1: `[3,4]` → 整数 {3,4}
   - 段2: `[4,5]` → 整数 {4,5}
   注意整数 3 同时属于段0和段1，整数 4 同时属于段1和段2，产生重叠。
3. 模拟涂色：
   - 操作 `[1,4]`：`left = mp[1]=0`，`right = mp[4]=2`。覆盖段0,1,2 → 所有段涂白。
   - 操作 `[3,5]`：`left = mp[3]=1`，`right = mp[5]=3`。覆盖段1,2 → 段1、段2涂黑。
4. 最终状态：
   - 段0: 白 (覆盖整数1,2,3)
   - 段1: 黑 (覆盖整数3,4)
   - 段2: 黑 (覆盖整数4,5)
   此时整数3 同时是白色（来自段0）和黑色（来自段1），矛盾！因为一个整数只能有一种颜色。问题根源在于段重叠导致同一个整数被多个段记录，且覆盖时无法统一。

**左闭右开如何解决**：
- 改用左闭右开 `[L, R+1)`，收集 `{1,5,3,6}` → `p = [1,3,5,6]`。
- 段定义：`[1,3)`、`[3,5)`、`[5,6)`，每个整数只属于一个段。
- 涂色：`[1,5)` 覆盖段0、1（白），`[3,6)` 覆盖段1、2（黑），最终段0白，段1黑，段2黑。白色段只有段0，对应整数1,2，正确。

这个例子清晰地展示了闭区间离散化因段重叠导致的颜色冲突，而左闭右开通过不重叠的段完美模拟了涂色过程。

## 四、总结对比

| 维度          | 一维（Color the Ball） | 二维（City Planning）             |
| ------------- | ---------------------- | --------------------------------- |
| 被覆盖对象    | 整数点（离散）         | 连续平面区域                      |
| 边界线性质    | 相邻整数点间距为1      | 边界线面积为0                     |
| 离散化表示    | 左闭右开 `[L, R+1)`    | 闭区间 `[L, R]`                   |
| 长度/面积计算 | `p[i+1] - p[i]`        | `(xs[i+1]-xs[i])*(ys[j+1]-ys[j])` |
| 部分覆盖问题  | 不会发生（边界对齐）   | 不会发生（边界对齐）              |
| 实现复杂度    | 低                     | 低                                |
| 适用场景      | 整数点区间覆盖         | 连续区域矩形覆盖                  |

### 核心思想
- 一维中，使用左闭右开将整数点转化为连续区间段，使得所有操作边界与段边界对齐，避免拆分段和重叠冲突。
- 二维中，使用闭区间即可，因为连续区域边界线面积为0，且坐标本身就是整数，差值直接得到宽度/高度，无需转换。

这种选择本质上是根据问题的离散性（整数点 vs 连续区域）和边界处理的需求来决定的。掌握这个区别，有助于在类似问题中选择合适的离散化方法。
