## P2652 同花顺 解题笔记

### 题目描述
有 \(n\) 张扑克牌，每张牌有花色 \(a_i\) 和数字 \(b_i\)。你可以任意更换一些牌（花色和数字都可随意改）。问最少更换多少张牌，能使最终所有牌构成一个**同花顺**，即所有牌花色相同，且数字是连续的整数（例如 \(1,2,3,4,5\)）。

### 关键点
- 最终花色必须一致。
- 最终数字必须互异且连续，共 \(n\) 个不同的整数，因此一定是一个长度为 \(n\) 的整数区间 \([L, L+n-1]\)。
- 最少更换 = **总牌数 \(n\) - 最多能保留的牌数。**

---

## 解题思路

### 1. 转化问题
选择一种花色作为最终花色。对于该花色下的所有牌，我们希望能保留尽可能多的牌，使得这些牌的数字可以全部放入某个长度为 \(n\) 的整数区间内（因为最终我们需要 \(n\) 个连续数字）。  
这等价于：在排序去重后的数字列表中，找到一个最长的子数组，使得**最大值与最小值之差 < n**。

### 2. 为什么是 < n 而不是 ≤ n？
- 区间长度为 \(n\)，例如区间 \([L, L+n-1]\) 包含 \(n\) 个整数。
- 如果保留的数字最小值为 \(x\)，最大值为 \(y\)，则 \(y-x \le n-1\) 才能放入该区间（因为 \(x\) 到 \(y\) 占用 \(y-x+1\) 个位置，不能超过 \(n\)）。  
- 条件 \(y - x < n\) 等价于 \(y - x \le n-1\)，这正是我们需要的。

### 3. 算法步骤
1. 读入 \(n\) 和所有牌。
2. 用哈希表（unordered_map）按花色分组，每组存储数字列表。
3. 对每组：
   - 排序。
   - 去重（同花色同数字的牌最多只能保留一张）。
   - 使用**滑动窗口**（双指针）在排序后的数组中找出满足 **最大值 - 最小值 < n** 的最长连续子数组长度，记为 `keep`。
4. 记录所有花色中最大的 `keep`，记为 `max_keep`。
5. 答案 = \(n - max_keep\)。

### 4. 滑动窗口详解
- 数组已排序，用指针 `l` 和 `r` 表示窗口左右边界。
- 遍历 `r` 从 0 到 `size-1`：
  - 当 `nums[r] - nums[l] >= n` 时，说明当前窗口极差过大，需要将左指针右移，直到条件满足。
  - 此时窗口内所有数字的极差 < n，它们可以全部保留。窗口大小 `r-l+1` 就是当前可保留的数量，更新最大值。
- 时间复杂度 \(O(k)\)，其中 \(k\) 为该花色的不同数字个数。

---

## 为什么需要去重？
- 最终同花顺中数字不能重复，所以同一花色中相同数字的多张牌只能保留一张。
- 去重后，滑动窗口才能正确反映可保留的最大数量。

## 错误分析（针对用户最初代码）
### 用户代码常见错误
1. **硬编码最终长度为 5**  
   `return max(5 - max_len, 0)` 导致 \(n \neq 5\) 时答案全错。

2. **要求数字严格连续（差 1）**  
   `if (arr[i] != arr[last] + i - last)` 只识别形如 `x, x+1, x+2,...` 的序列，而实际只需要极差 < n，不要求中间无空缺。

3. **答案计算逻辑错误**  
   取 `min_need` 虽然等价于取最大保留数，但前提是 `need_card` 能正确计算保留数。由于保留数计算错误，最终结果必然错误。

4. **未处理 \(n=0\) 特殊情况**  
   （虽然题目 \(n\) 可以为 0，但测试点可能包含）

## 正确代码（C++）

```cpp
#include <iostream>
#include <vector>
#include <unordered_map>
#include <algorithm>
using namespace std;

int main() {
    int n;
    scanf("%d", &n);
    if (n == 0) {
        printf("0\n");
        return 0;
    }
    unordered_map<int, vector<int>> groups;
    for (int i = 0; i < n; ++i) {
        int a, b;
        scanf("%d%d", &a, &b);
        groups[a].push_back(b);
    }

    int max_keep = 0;
    for (auto &p : groups) {
        vector<int> &nums = p.second;
        sort(nums.begin(), nums.end());
        nums.erase(unique(nums.begin(), nums.end()), nums.end());
        int k = nums.size();
        int l = 0;
        for (int r = 0; r < k; ++r) {
            while (nums[r] - nums[l] >= n) {
                ++l;
            }
            max_keep = max(max_keep, r - l + 1);
        }
    }
    printf("%d\n", n - max_keep);
    return 0;
}
```

---

## 复杂度分析
- 时间：排序总复杂度 \(O(n \log n)\)，滑动窗口 \(O(n)\)，总 \(O(n \log n)\)。
- 空间：存储所有牌 \(O(n)\)。

---

## 总结
本题的关键在于将“最少更换”转化为“最多保留”，并利用滑动窗口在排序去重后的数字列表中高效求出满足极差条件的最长子数组。理解了这一点，代码就非常简洁。
