# 左闭右开区间 vs 闭区间：对比笔记

在处理区间覆盖问题时，选择用 **闭区间 `[L, R]`** 还是 **左闭右开区间 `[L, R+1)`** 会直接影响离散化、涂色模拟和长度计算的复杂度。下面从几个维度进行对比。

---

## 1. 区间表示

| 表示法              | 含义                                    | 包含整数     |
| ------------------- | --------------------------------------- | ------------ |
| 闭区间 `[L, R]`     | 从 L 到 R 的所有整数                    | L, L+1, …, R |
| 左闭右开 `[L, R+1)` | 从 L 到 R 的所有整数（因为 R+1 不包含） | 与闭区间相同 |

**区别**：  
- 闭区间直接对应题目给出的输入。  
- 左闭右开是对原始输入的一种转换，其右端点比原右端点大 1。

---

## 2. 离散化

假设有若干操作 `[L_i, R_i]`，我们需要收集关键点用于离散化。

| 表示法              | 收集的关键点        | 离散化后的段                                                                                                        |
| ------------------- | ------------------- | ------------------------------------------------------------------------------------------------------------------- |
| 闭区间 `[L, R]`     | 每个操作的 L 和 R   | 相邻点之间的区间 `[p[i], p[i+1]]`，长度 `p[i+1] - p[i] + 1`。但相邻段可能重叠或不连续，需要额外判断。               |
| 左闭右开 `[L, R+1)` | 每个操作的 L 和 R+1 | 相邻点之间的区间 `[p[i], p[i+1])`，长度 `p[i+1] - p[i]`。相邻段自然衔接（右端点等于下一段的左端点），无需判断空隙。 |

**分析**：  
- 闭区间离散化时，相邻点之间可能间隔多个整数，也可能恰好相邻。例如两个操作 `[1,4]` 和 `[5,8]`，闭区间离散化点 {1,4,5,8}，段1为 `[1,4]`（长度4），段2为 `[5,8]`（长度4），中间没有空隙，但无法通过点直接看出 `4` 和 `5` 之间没有缺失整数（因为 `4+1=5`）。如果存在空隙，比如 `[1,2]` 和 `[4,5]`，闭区间离散化点 {1,2,4,5}，段 `[1,2]` 长度2，段 `[4,5]` 长度2，中间的空隙（整数3）没有对应段，这会导致覆盖时无法标记中间部分。  
- 左闭右开时，`[1,3)` 和 `[3,6)` 自然衔接，不会产生间隙。即使有间隙，如 `[1,3)` 和 `[4,6)`，点集为 {1,3,4,6}，段 `[1,3)` 和 `[4,6)` 之间就有 `3` 到 `4` 的间隙，这个间隙可以通过 `points` 的差值直接体现（`3` 到 `4` 差值为1，代表整数3未覆盖）。

---

## 3. 覆盖操作

| 表示法   | 覆盖操作                                                                                                                                                                                                                                             |
| -------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 闭区间   | 需要找到 L 和 R 对应的离散化下标 `left` 和 `right`，然后覆盖 `[left, right]` 的所有段。但注意，段 `[p[i], p[i+1]]` 覆盖的整数是连续的，当覆盖 `[left, right]` 时，可能需要将段分裂成更小的部分（如果覆盖范围只占原段的一部分）。这通常需要额外处理。 |
| 左闭右开 | 找到 L 和 R+1 对应的离散化下标 `left` 和 `right`，然后覆盖 `[left, right)` 的所有段。因为每个段对应一个连续整数区间，覆盖时不会出现部分覆盖的情况（如果原段完全在操作内部，就整个覆盖；如果操作边界与段边界对齐，则覆盖连续几个整段）。不需要分裂。  |

**分析**：  
- 闭区间覆盖时，如果操作边界落在某个段的中间，就必须将该段拆分成两部分，分别涂色。这需要更复杂的处理。  
- 左闭右开通过将每个操作的右端点 +1，使得所有操作的边界都落在离散化点上（因为离散化时已经包含了这些点），因此每个操作覆盖的总是若干完整的段，无需分裂。

---

## 4. 长度计算

| 表示法   | 计算一个白色段的总长度                                                                                       | 输出右端点                                                                                            |
| -------- | ------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------- |
| 闭区间   | 段 `[p[i], p[i+1]]` 的长度 = `p[i+1] - p[i] + 1`。求和后需小心相邻段是否连续，连续则累加，否则可能多算空隙。 | 右端点 = 最后一个段覆盖的最后一个整数，即 `p[right_index+1]`（如果段索引为 `i`，右端点为 `p[i+1]`）。 |
| 左闭右开 | 段 `[p[i], p[i+1])` 的长度 = `p[i+1] - p[i]`。直接累加，因为相邻段自动衔接（右端点等于下一段左端点）。       | 右端点 = 最后一个段覆盖的最后一个整数 = `p[right_index+1] - 1`（因为段是左闭右开）。                  |

**分析**：  
- 左闭右开避免了判断相邻段是否连续的麻烦，因为段之间的间隙已经体现在 `points` 的差值中（若差值为1，则中间没有整数；若差值>1，则中间有整数未被覆盖）。扫描时只需按顺序累加长度，无需额外逻辑。  
- 闭区间需要额外维护每个段的长度公式，且当遇到间隙时需要正确处理（例如跳过间隙）。

---

## 5. 代码复杂度

| 表示法   | 实现难度                                             | 常见错误                                                |
| -------- | ---------------------------------------------------- | ------------------------------------------------------- |
| 闭区间   | 较高，需要处理段分裂、长度计算中的 +1/-1、间隙判断等 | 容易在边界处理上出错，如漏掉最后一个整数、长度计算少1等 |
| 左闭右开 | 较低，只需收集 L 和 R+1，覆盖整段，长度直接相减      | 容易忘记对 R+1 的处理，或在输出右端点时忘记减1          |

---

## 6. 适用场景

- **闭区间**：适用于问题本身不要求精确处理间隙，或者区间操作保证连续且不会部分覆盖时。但在本题中，由于操作顺序任意，且可能覆盖任意子区间，闭区间实现较繁琐。
- **左闭右开**：是处理区间覆盖问题的**标准技巧**，尤其适合离散化 + 扫描的解法。很多经典问题（如涂色、区间合并、扫描线）都采用这种表示法，因为它能统一处理边界，减少出错。

---

## 7. 总结

**推荐使用左闭右开 `[L, R+1)` 表示法**，原因如下：

- **离散化简单**：只需收集 L 和 R+1，无需额外考虑 +1/-1。
- **覆盖不分裂**：每个操作覆盖若干完整的离散化段，无需处理部分覆盖。
- **长度计算直观**：段长度 = `p[i+1] - p[i]`，直接累加，无需处理 +1 或间隙判断。
- **输出端点明确**：左端点 = `points[start]`，右端点 = `points[end+1] - 1`。

**记忆要点**：
- 将题目给的闭区间 `[a, b]` 在离散化时转换为 `[a, b+1)`。
- 所有涉及边界的地方都使用左闭右开：离散化点的选择、覆盖范围、长度累加。
- 最终输出时，右端点要减 1（因为开区间不包含右端点）。

掌握了这个技巧，就能轻松应对类似的区间覆盖问题。
